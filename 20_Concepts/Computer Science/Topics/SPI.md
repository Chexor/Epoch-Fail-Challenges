# SPI (Serial Peripheral Interface)
#concept #cs #embedded #communication

## 1. Wat is het kernprobleem?
How can a microcontroller communicate with multiple high-speed peripherals (like sensors, displays, or memory) using a minimal number of wires while maintaining high data throughput?

## 2. Intuitieve uitleg
SPI werkt als een team waarbij één persoon (de **Master**) het tempo bepaalt met een metronoom (de **Clock**). Terwijl de Master spreekt tegen een teamlid (via **MOSI**), kan dat teamlid tegelijkertijd iets terugzeggen (via **MISO**). De Master tikt met zijn vinger op de schouder van het teamlid (**Chip Select**) om te laten weten: "Nu praat ik met jou."

## 3. Formele structuur
SPI is a synchronous, four-wire serial bus.

### The Four Lines:
1. **SCLK (Serial Clock)**: Pulse train generated by the master to synchronize data bits.
2. **MOSI (Master Out Slave In)**: Data line for sending data from master to peripheral.
3. **MISO (Master In Slave Out)**: Data line for receiving data from peripheral to master.
4. **CS / SS (Chip Select / Slave Select)**: Active-low signal used to enable a specific peripheral.

### Characteristics:
- **Full-Duplex**: Data can be sent and received simultaneously.
- **Synchronous**: Uses a clock signal (unlike UART).
- **High Speed**: Often supports frequencies in the MHz range.
- **Master-Slave Architecture**: One master controls multiple slaves.

## 4. Toepassing
Reading data from an SPI temperature sensor:
1. Pull the sensor's **CS** pin LOW.
2. Send the "Read Temperature" command byte via **MOSI**.
3. Simultaneously receive the temperature data byte via **MISO**.
4. Pull the **CS** pin HIGH to end communication.

## 5. Examengerichte vertaling
- **Typische vraag:** "Wat is het verschil tussen SPI en I2C qua snelheid en bekabeling?"
- **Vaak gemaakte fout:** Vergeten dat SPI full-duplex is; je ontvangt altijd een byte als je er een verstuurt (ook als dat een dummy-byte is).
