# Object-Oriented Design Principles
#concept #sdp

Object-Oriented Design Principles groepeert richtlijnen die helpen om objectgeoriënteerde software onderhoudbaar, uitbreidbaar en begrijpelijk te maken.

## Korte kern

- vormt de kapstok voor de SOLID-principes
- helpt ontwerpkwaliteit systematisch beoordelen
- maakt code robuuster tegen verandering

## 1. Wat is het kernprobleem?
Hoe ontwerp je objectgeoriënteerde software zo dat ze minder fragiel, beter uitbreidbaar en makkelijker te onderhouden is?

## 2. Intuitieve uitleg
Wanneer code groeit, wordt slecht ontwerp snel pijnlijk: wijzigingen lekken door, verantwoordelijkheden lopen door elkaar en afhankelijkheden worden hard. Ontwerpprincipes helpen die problemen voorkomen.

## 3. Formele structuur
Belangrijke principes in deze groep zijn:

- [[S - Single Responsibility Principle]]
- [[O - Open-Closed Principle]]
- [[L - Liskov Substitution Principle]]
- [[I - Interface Segregation Principle]]
- [[D - Dependency Inversion Principle]]

## 4. Snelle vergelijking

- [[Object-Oriented Design Principles]] = overkoepelende ontwerpregels
- [[SOLID Principes]] = map-note en leerkaart voor deze principes

## 5. Toepassing
Wanneer je een klasse- of interfacestructuur evalueert op onderhoudbaarheid, gebruik je vaak precies deze ontwerpprincipes als kapstok.

## 6. Begripsafbakening en verbanden
- **Kernonderscheid:** dit is een inhoudelijke paraplunote voor ontwerpprincipes, niet enkel een navigatienote.
- **Veelvoorkomende misvatting:** SOLID zien als vijf losse slogans zonder gemeenschappelijke ontwerpambitie.
