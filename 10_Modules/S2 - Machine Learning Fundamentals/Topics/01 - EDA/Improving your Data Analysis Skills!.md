# Improving your Data Analysis Skills!

Although this course focuses on machine learning algorithms and models, it is important to remember that model training is always preceded by an **Exploratory Data Analysis (EDA)**. During the EDA phase, the training data are explored, cleaned, preprocessed, and transformed. The primary purpose of EDA is to gain **insight into the data**: understanding its structure, identifying patterns, detecting anomalies, and assessing data quality.

EDA relies heavily on techniques from descriptive statistics to explore, summarize, and visualize the dataset. It is typically carried out in **Pandas**, a Python library for data manipulation. In addition, you will use plotting libraries—such as **Matplotlib**, **Seaborn**, or **Plotly**—to create the informative statistical visualizations needed to better understand the data.

After the EDA phase, the next step is **feature engineering**, where you transform, combine, or create new features that will serve as inputs for your machine learning models. Some practitioners consider feature engineering as part of EDA, while others treat it as a separate step that follows the exploratory phase. In this course, we treat EDA primarily as a stage for understanding and diagnosing the data, before moving on to feature engineering and model development.

This topic of the course focuses on Exploratory Data Analysis (EDA) and contains the **following materials**:

- A **summary presentation** introducing the key ideas behind EDA.
- The **textbook** _Practical Statistics for Data Scientists, 2nd Edition_ by Peter Bruce, Andrew Bruce & Peter Gedeck (2020). You only need to go through **Chapter 1**, as it is fully dedicated to EDA.
- The **notebook** and datasets that accompany Chapter 1 of the textbook.
- An **EDA exercise**, provided as a notebook both _with_ and _without_ solutions, along with the required dataset in CSV format.
- An **EDA challenge**, which you must solve in groups and submit before the deadline.

This topic is not the primary focus of the course, as it is assumed that you already have a sufficient background in descriptive statistics, data analysis, and Pandas. However, since EDA always comes **_before_ the modelling phas**e, it is essential that you understand what EDA is and why it is used.

Most of the machine learning exercises in this course will rely on datasets that have already been cleaned for you. Nevertheless, you should still be able to **perform a basic EDA** yourself before training models, as this is a crucial step in any real‑world data science workflow.
