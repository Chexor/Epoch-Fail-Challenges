# Les 4

## Command Pattern
- Een opdracht wordt verpakt in een apart object.
- Zo kan je een actie loskoppelen van wie ze uitvoert.

## Strategy Pattern
- Verschillende algoritmes worden apart verpakt achter dezelfde interface.
- Zo kan je tijdens runtime kiezen welke aanpak je gebruikt.

## Iterator Pattern
- Met een iterator doorloop je elementen van een collectie stap voor stap.
- Je kan over de data itereren zonder de interne structuur van de collectie te kennen.


Wat is Runtime?
- Runtime is het moment waarop een programma echt uitgevoerd wordt.
- Dan draait de code, worden objecten aangemaakt en gebeuren berekeningen of acties effectief.

DRY-principe
- DRY = Don't Repeat Yourself.
- Vermijd dubbele code of dubbele logica, zodat je wijzigingen maar op 1 plaats moet doen.
