# Overzicht Concepten
#concept #meta

Dit is de centrale ingang van `20_Domains/` met navigatie per domein en het laagmodel `Domein -> 01_Atlas -> 02_Theory -> 03_Atom`.

## Korte kern

- start hier als je snel het juiste kennisdomein zoekt
- gebruik `01_Atlas` voor navigatie, `02_Theory` voor uitleg, `03_Atom` voor compacte termen
- gebruik Bases als operationele indexlaag bovenop de inhoudelijke structuur

## 1. Wat is het kernprobleem?

Hoe navigeer je snel in een groeiende kennisbank zonder dubbele notes, vage plaatsing of mix van note-rollen?

## 2. Intuitieve uitleg

`20_Domains/` is je duurzame kennislaag. Je ordent niet per cursus, maar per kennisdomein. Binnen elk domein hou je rollen zuiver:

- `01_Atlas`: overzicht, leerpad, conceptkaart
- `02_Theory`: volledige conceptuitleg
- `03_Atom`: korte term-definitie

`05_Versus/` en `04_Practice/` blijven aanvullende lagen voor respectievelijk contrast en toepassing.

## 4. Snelle vergelijking

- `01_Atlas` = didactische navigatie
- `02_Theory` = kernuitleg van concepten
- `03_Atom` = snelle begripsdefinities
- `05_Versus` = expliciete conceptverschillen
- `04_Practice` = worked examples
- `Bases` = filter-, zoek- en kwaliteitslaag

## 5. Toepassing

Typische flow:

1. start in een domeinoverzicht
2. ga via `01_Atlas` naar de juiste `02_Theory`-note
3. open `03_Atom` voor snelle begripsverduidelijking
4. gebruik `05_Versus` of `04_Practice` als je contrast of praktijk nodig hebt

## 7. Praktische gebruiksregels

- start bij twijfel in de domein-`01_Atlas`
- schrijf nieuwe conceptuitleg in `02_Theory/`
- gebruik `03_Atom/` enkel voor echt atomische term-notes
- plaats IoT-specifieke hardware- en protocolconcepten in `20_Domains/IoT/`
- voeg nieuwe domeinen consistent toe in zowel sectie `3. Formele structuur` als sectie `6. Domeinen`
- hou metadata consistent (`type`, `domain`, `parent`, `related`, `theme`, `aliases`)
