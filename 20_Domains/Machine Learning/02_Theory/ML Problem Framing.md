# ML Problem Framing
#concept #ml

ML Problem Framing groepeert de concepten die bepalen welk soort machine-learningprobleem je precies probeert op te lossen.

## Korte kern

- helpt bepalen wat input, output en probleemtype zijn
- vormt de kapstok voor features, targets en classification vs regression
- komt vóór modelkeuze

## 1. Wat is het kernprobleem?
Hoe bepaal je eerst correct welk probleem je oplost voor je een model of algoritme kiest?

## 2. Intuitieve uitleg
Voor je een model kiest, moet je weten wat je input is, wat je wil voorspellen en welk type taak je voor je hebt. Dat voorbereidende denkwerk is problem framing.

## 3. Formele structuur
Belangrijke concepten binnen deze kapstok zijn:

- [[Data, Example, Feature, Target]]
- [[Classification vs Regression]]

Deze bepalen:

- welke variabelen je invoert
- wat als target telt
- of je te maken hebt met classificatie, regressie of clustering

## 4. Snelle vergelijking

- [[ML Problem Framing]] = probleemtype juist afbakenen
- [[Learning Paradigms]] = soort leerlogica bepalen
- [[Linear Modeling]] = concreet type model kiezen

## 5. Toepassing
Bij een dataset met woningkenmerken en prijzen moet je eerst zien dat `prijs` de target is en dat je dus een regressieprobleem hebt. Dat is ML problem framing.

## 6. Begripsafbakening en verbanden
- **Kernonderscheid:** problem framing gaat over het probleem juist formuleren, niet over het model trainen.
- **Veelvoorkomende misvatting:** direct een algoritme kiezen zonder eerst het probleemtype scherp te hebben.
